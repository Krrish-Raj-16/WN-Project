### Core behavior
| Piece | Implementation |
|--------|------------------|
| **Tasks** | Permuted MNIST/FMNIST (`task_type: permuted`) and split CIFAR-10 (`task_type: split`, disjoint class groups). |
| **Non-IID** | Dirichlet labeling (`dirichlet_partition` / alias `partition_data`); `alpha: "iid"` or a float. |
| **Models** | `MLP` (784→400→400→10) and `SmallCNN` for CIFAR; `get_parameters` / `set_parameters` in `model.py`. |
| **EWC** | `compute_fisher`, `EWC` class with `consolidate` + `penalty` in `ewc.py`. |
| **Clients** | `FedAvgClient` and `EWCClient` in `client.py` (task loss + `ewc.penalty`). |
| **Server** | Flower `FedAvg` + centralized `evaluate_fn` on **per-task global test loaders**; fit config supplies `task_id` and **`finalize_task`** (true at the **last round of each task segment**) so **`consolidate` runs after training that task**, matching the paper (Fisher after task completion). This replaces the doc’s `is_new_task` snippet, which would store Fisher for the wrong task if taken literally. |
| **Metrics** | CSV logger + `*_summary.json` with CFR between task boundaries and BWT. |
| **Simulation** | `requirements.txt` uses **`flwr[simulation]`** (Ray) so `start_simulation` works. |
